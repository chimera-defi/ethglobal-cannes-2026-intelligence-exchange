## Intelligence Exchange Alternatives and Variants

## Core Alternatives
1. Router-only SaaS (no marketplace)
   - Lower operational complexity
   - Less defensibility and weaker liquidity network effects
2. Compute marketplace only (no policy-aware routing)
   - Easier to launch supply side
   - Harder to win buyer trust for production workloads
3. Enterprise private exchange (single-tenant)
   - Higher ACV potential
   - Slower sales cycles and integration burden

## Product Variants
1. Verticalized exchange for agentic customer support workloads.
2. Fintech-safe policy profile exchange for regulated teams.
3. Internal enterprise exchange for multi-team budget governance.
4. Cannes 2026 agentic idea-build marketplace (`CANNES_2026_VARIANT.md`).

## Commercial Variants
1. Take-rate only
2. Subscription + lower take-rate
3. SLA tiers with premium reliability guarantees

## Payment Variants
1. Stripe-only start (fastest to launch)
2. Stripe + ACP-style agent payments
3. Add crypto settlement (Strike-style/BTC rail) after mainstream billing proves traction

## Recommendation
Start with:
1. Router + exchange hybrid
2. Stripe-first billing
3. Add ACP/crypto rails as optional expansion, not as core dependency

Hackathon-specific variant:
1. Narrow to idea -> brief -> build milestone flow
2. Use Arc + World + 0G as the primary sponsor stack
3. Keep ENS and Ledger as additive trust layers only if they remain native to the demo
