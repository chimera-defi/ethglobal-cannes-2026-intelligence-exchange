## Forge / Foundry Rules

**IMPORTANT: warnings ≠ errors.**
`forge build` emits `warning[erc20-unchecked-transfer]` and similar warnings. These are NOT blocking. Only lines starting with `error[` are compilation errors. Always run `~/.foundry/bin/forge test ...` after making contract changes even if warnings are present.

**Foundry binary:** `~/.foundry/bin/forge` (not in PATH by default).

**Mandatory test run after any contract change:**
```bash
~/.foundry/bin/forge test --match-contract 'ContractA|ContractB' 2>&1 | tail -30
```
Report: exact pass/fail count. "Syntactically correct" is NOT a substitute.

## Common Test Pitfalls

1. **Parametric error selectors** — `vm.expectRevert(Error.selector)` fails for errors with parameters if the revert data is longer than 4 bytes. Use: `vm.expectRevert(abi.encodeWithSelector(Error.selector, param1, param2))`.

2. **TWAP staleness** — `vm.warp()` by > 2 hours stales the TWAP. Any test calling `executeMint()` after a long warp must first call `controller.updateTWAP(INITIAL_TWAP)`.

3. **Flow bonus timing** — The flow bonus (P16B-M2 fix) requires `block.timestamp >= flowBonusEligibleAt` (set to `stake time + 1 day`). Tests checking mint allowance bonuses need `vm.warp(block.timestamp + 1 days + 1)`.

4. **StakerInfo struct fields** — After P16B-M2 fix, `StakerInfo` has 11 fields. Any code tuple-unpacking it (e.g., DisputeResolution) must be updated to handle 11 fields.
